# Codeword Set — canonical specification (Canonical Baseline)

## Purpose

This specification defines the canonical **codeword set** `C ⊂ F2^9` for the ASH Pattern System.

The codeword set is the algebraic structure that governs all state transformations, orbit structure, averaging behavior, and branching topology in the ASH Pattern System. Every transformation `x' = x ⊕ c` uses a codeword from this set.

## Closure status

> **STATUS: FULLY CLOSED**
>
> The canonical codeword set `C` is fully specified below. Exact generators, exhaustive enumeration, and code parameters are part of this repository's canonical baseline. There are no remaining open closure items for the codeword set.

## Canonical basis

The canonical codeword set is defined by six basis-reference vectors, of which four are linearly independent:

- `(1, 1, 1, 1, 0, 0, 0, 0, 0)` — weight 4
- `(1, 1, 0, 0, 1, 1, 0, 0, 0)` — weight 4
- `(1, 0, 1, 0, 1, 0, 1, 0, 0)` — weight 4
- `(1, 0, 0, 1, 1, 0, 0, 1, 0)` — weight 4
- `(1, 1, 1, 1, 1, 1, 1, 1, 0)` — weight 8
- `(0, 0, 0, 0, 1, 1, 1, 1, 0)` — weight 4

### Algebraic closure

Rank analysis over `F2` shows the 6 vectors have rank 4. Two are linearly dependent: `g5 = g1 ⊕ g6`. The 4 independent vectors (g1–g4) form a basis for a 4-dimensional subspace of `F2^9`.

XOR-closure computation from the 4 independent generators produces exactly 16 = 2^4 vectors. This is a deterministic algebraic consequence — given the generators, the complete subgroup is uniquely determined.

### Resulting canonical codeword set

The 16-member set produced by XOR-closure of the generators is the canonical codeword set `C`. Its properties ([9, 4, 4] doubly-even, subgroup, 32 orbits) are verified by exhaustive enumeration below.

## Structural properties

### Subgroup

`C` is a subgroup of `(F2^9, ⊕)`: it contains the identity (zero vector), and is closed under XOR. This is confirmed by exhaustive enumeration and generator closure.

### Ninth-coordinate property

Every codeword in `C` has `b8 = 0` (the 9th coordinate is zero for all 16 members). This is a verified structural fact of the complete canonical code.

This property is treated as an algebraic fact of the canonical codeword set embedded in `F2^9`. All 9 coordinates remain structurally equal participants in the state space at the foundational level.

## Downstream implementation constraints

### Implementations must

- Use exactly the 16-codeword set specified above — no more, no fewer
- Validate that the provided codeword set matches this specification
- Treat `C` as a fixed, canonical constant — not a runtime parameter or implementation choice
- Enforce that all codewords used in transformations are members of `C`

### Implementations must not

- Invent codewords not in the canonical set
- Extend `C` beyond the 16 specified members
- Treat the codeword set as an open parameter that implementations may choose freely
- Infer extra coordinate-derivation rules from the `b8 = 0` property
